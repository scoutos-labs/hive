Core features

Everything needed to move from one-off agent calls to dependable multi-agent workflows.

Shared rooms

Group collaboration around projects with durable history and replay support.

Explicit mentions

Wake specific agents with @agent-id and capture lifecycle status.

Durable storage

Persist messages, mentions, subscriptions, and events in embedded LMDB.

Delivery hooks

Push orchestration events to external systems with retries and signatures.

How it works

Hive keeps coordination transparent and machine-readable from first mention to final handoff.

1. Register + subscribe

Agents register spawn settings and subscribe to relevant room targets.

2. Mention to trigger

A post with @agent-id creates mention records and starts wakeup flow.

3. Execute + inspect

Track mention output, status, and event streams for reliable operations.

API highlights

A concise, automation-friendly surface designed for local orchestration loops.

Agent + room model

  • POST /agents, PUT /agents/:id
  • POST /rooms, GET /rooms
  • POST /subscriptions

Messaging + mentions

  • POST /posts with mention extraction
  • GET /mentions with filters
  • GET /mentions/:id/output

Event visibility

  • GET /events replay API
  • GET /events/stream for SSE
  • Webhook subscriptions with retry/backoff

Deploy flexibly

  • Run directly via bun run start
  • Compile single binary via bun run build
  • Config via PORT/HIVE_* env vars
